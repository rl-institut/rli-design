RMS Design System

Layout

RMS is primarily based on ZURB Foundation. We will migrate the CSS system to Bootstrap. In order to prevent display errors and bugs, we should be sure we don't use classes that are used in Foundation. Since Bootstrap has the same type of structure as Foundation, we should use a different system. For that purpose, use BEM system as much as possible: block__element--modifier. Whenever possible use @extend from Sass: @extend {bootstrap-class} if the styles you are looking for are already available under a Bootstrap class.

There are exceptions to the use of BEM, such as .row and .col. Use in that case the .bs-row and .bs-col classes. @extend is used to integrate the Bootsrap class declarations in these custom classes. Similarly, if only a Bootstrap class can be used and not the BEM system, create a new class and declaration: .bs-{bootstrap-class} and extend the Bootstrap class: @extend {bootstrap-class}.

To override Bootstrap variables, use _variable_overrides.scss and place it at the right position in the document (check where it is located inside node_modules/bootstrap/scss/_variables.scss), to be sure not to override the varibale more than once.

Colors

Primary color is $blue. It is mostly used for text, but also some background elements images.

Color for anchors is $link-color.

Try to use very light background colors such as $white or $gray-100. You can use box-shadows such as $box-shadow-sm on a component if the component and background both have $white as a background-color.

Design tokens

With design tokens we want to create a clear design structure that can be used by both designers and developers across projects. They are still in test phase and need be updated but they already can be used. To know more about design tokens, please read this article from InVision.

The tokens are constructed as following: category.usage.component.subitem.state whereas subitem and state are optional.

Since our CSS structure heavily relies on @extend in order to properly combine BEM and Bootstrap, the tokens are related to both Bootstrap CSS variables and utility classes.
